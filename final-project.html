Final Project

Weight: 30% of final grade
Team Size: 2-3 students (individual projects possible with approval)
Duration: Developed progressively throughout the semester
Final Presentation: December 7, 2025 (15 minutes per team)

Your final project should demonstrate mastery of neuroimaging/neurostimulation concepts through original research, device development, or software creation. Projects are integrated throughout the course, not a last-minute assignment.

Data Analysis Project

Design and conduct an original data analysis addressing a specific research question using neuroimaging data.

What you'll create:
  • Research question with literature review
  • Data collection protocol (in-person or online)
  • Complete analysis pipeline with statistical validation
  • Scientific report with publication-quality figures
  • Reproducible code and documentation

Examples: ERP study of attention, fMRI connectivity in meditation, EEG-based emotion recognition, mobile EEG during physical activity

Device Design Project

Design and build a novel neuroimaging or neurostimulation device, or significantly improve an existing system.

What you'll create:
  • Engineering specifications and design rationale
  • Prototype device (physical or detailed simulation)
  • Safety analysis and regulatory considerations
  • Performance validation with test data
  • User manual and technical documentation

Examples: Low-cost EEG amplifier, portable tDCS device, fNIRS probe optimization, artifact reduction hardware

Software Development Project

Develop software tools, applications, or platforms that advance neuroimaging research or clinical applications.

What you'll create:
  • Functional software application with user interface
  • Technical documentation and API reference
  • Validation with real data and user testing
  • Open-source code repository with clear documentation
  • Deployment guide and installation instructions

Examples: Real-time BCI application, EEG analysis web app, VR neurofeedback system, automated preprocessing pipeline

Example Project Ideas

Data Analysis Projects

  • Mobile EEG during Exercise: Compare alpha/beta rhythms during stationary vs. cycling conditions
  • Meditation and Connectivity: fMRI analysis of default mode network changes during mindfulness
  • Music and Emotion: EEG responses to different musical genres using affective computing
  • Sleep Stage Classification: Machine learning approach to automated sleep scoring from EEG
  • fNIRS Language Study: Hemispheric activation patterns during bilingual word processing

Device Design Projects

  • Arduino-based EEG: Low-cost 8-channel EEG system with smartphone app
  • Wearable tDCS: Portable stimulation device with safety monitoring
  • fNIRS Probe Optimization: 3D-printed optode holders for improved signal quality
  • Artifact Reduction Hardware: Active shielding system for mobile EEG
  • Hybrid EEG-fNIRS: Combined system for simultaneous electrical and optical recording

Software Development Projects

  • Real-time BCI Speller: Web-based SSVEP typing interface
  • EEG Analysis Dashboard: Interactive visualization tool for preprocessing and analysis
  • VR Neurofeedback: Immersive environment controlled by real-time EEG
  • Mobile EEG App: Smartphone application for data collection and basic analysis
  • Automated Preprocessing: Machine learning pipeline for artifact detection and removal

Resources & Support

Equipment Available

  • 25 Muse EEG headsets for data collection
  • Access to campus MRI facility (with supervision)
  • fNIRS system for optical brain imaging
  • tDCS and TMS devices for stimulation projects
  • Electronics lab with Arduino, Raspberry Pi, 3D printer
  • High-performance computing resources for analysis

Software & Computing

  • Analysis: Python (MNE, SciPy), MATLAB (EEGLAB, SPM), R
  • Development: Web technologies (JavaScript, React), mobile (React Native, Flutter)
  • Hardware: Arduino IDE, KiCad for PCB design, Fusion 360 for 3D modeling
  • Computing: Google Colab Pro, university HPC cluster access

Mentorship

  • Weekly office hours with instructor and TAs
  • Individual team consultations throughout the semester
  • Access to graduate student mentors in related research areas
  • Industry connections for device design and software projects

Submission Requirements

Final Report (Due December 7)

  • Length: 8-12 pages (excluding references and appendices)
  • Format: Scientific paper style with abstract, introduction, methods, results, discussion
  • Figures: Publication-quality plots, diagrams, and photographs
  • Code: Complete, documented, and reproducible (GitHub repository preferred)
  • Data: Raw and processed data with clear documentation

Final Presentation (December 7)

  • Duration: 15 minutes presentation + 5 minutes Q&A
  • Format: Professional slides with live demonstration if applicable
  • Content: Problem motivation, approach, results, implications, future work
  • Audience: Classmates, instructor, invited guests from industry/academia

Ethics Note: All projects involving human participants must receive ethics approval. We'll work together to ensure proper protocols are followed for data collection.
